## 지시사항
"채팅 의도 패턴 관리" 의 내용은 지금 만든시스템을 사용하는 회사에 따라서 구분되어야해
XSUAA 과 IAS 를 통해 로그인될때 고객사를 구분하는 값을 참조해야 하는데 
어느값을 참조해야하는지 아직은 정해지지 않았으니, 
일단 모든 로그인한 사용자들은 
SK Networks 직원으로 설정이 되어야 하고, 

"채팅 의도 패턴 관리" 에 데이터 입력시 고객사 구분코드가 자동으로 할당 되어야해
DB 에 저장시도 고객사 구분코드가 저장되어야해

고객사구분코드는 
현재는 "SKN", "SKT", "SKI", "SKAX" 4개 구분코드만 있으면 되는데 
지금은 하드코딩으로 나타나면 되고, 
향후에는 더 추가될 예정이야

그리고 채팅시에는 로그인한 사용자의 고객사에 해당하는 채팅 의도 패턴만 적용 되어야해

"채팅 의도 패턴 관리" 에서 "패턴추가" 버튼을 누르면 고객사 구분코드를 선택하는 란이 ComboBox 형태로 제공이 되어야해
